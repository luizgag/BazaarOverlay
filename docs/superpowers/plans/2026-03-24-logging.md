# Logging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `Microsoft.Extensions.Logging` with console provider for development/debugging observability.

**Architecture:** Inject `ILogger<T>` into existing services via constructor DI. Console provider writes to an allocated debug console. No custom abstractions — standard .NET logging throughout.

**Tech Stack:** `Microsoft.Extensions.Logging`, `Microsoft.Extensions.Logging.Console`, `Microsoft.Extensions.Logging.Abstractions`, `NullLogger<T>` for tests.

**Spec:** `docs/superpowers/specs/2026-03-24-logging-design.md`
